Conceptual and documentation review skill for documentation-heavy cases.

Goal:
- Produce a defensible conceptual review when runnable artifacts are absent.
- Identify methodology gaps, document inconsistencies, data quality observations, and evidence still needed for execution-based validation.

Focus areas:
- Methodology coherence.
- Prior validation conditions and open evidence requests.
- Feature and reason-code documentation coverage.
- Data extract quality and monitoring readiness.
- Documentation-readiness benchmarking against a fixed validation rubric.

Execution posture:
- Runtime modules should remain blocked unless the evidence actually supports them.
- Use documentation and dataset tools aggressively before concluding that the package is only conceptual.
- A gateway sidecar documentation benchmark is acceptable for best-practice comparison, but it must not replace grounded review of the supplied artifacts.
